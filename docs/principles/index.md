## Why This Matters

These principles are the most common interview topics at every level. More importantly, they define the difference between code that is easy to change and code that accumulates technical debt.

> "Clean code always looks like it was written by someone who cares." — Robert C. Martin

---

## Core Takeaways

- **OOP** gives you tools: classes, interfaces, inheritance.
- **SOLID** tells you how to use those tools *well*.
- **KISS/DRY/YAGNI** remind you when to *stop* using them.

The goal is not to follow rules mechanically but to understand the tradeoffs they protect you from.
